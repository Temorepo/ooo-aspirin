Class ComparatorsMethods
 
Packagecom.threerings.util
Classpublic class Comparators

Contains sorting Comparators. These functions take any parameters but require only the first to, so they are suitable for passing to Array.sort(), or with a flex Sort object. Note that returning values other than 1, 0, or -1 confuse a flex Sort object. In general, these functions don't handle sorting undefined, it will be coerced to null.

compareUnknowns()method 
public static function compareUnknowns(o1:Object, o2:Object, ... ignored):int

Compare two values whose type is not known at compile type. Tries to figure it out. Also handles nulls. Note that this method may look like it can safely compare heterogeneous types, but it cannot. Consider the array: [ true, "meerkat", false ]. compareUnknowns(true, "meerkat") returns -1; compareUnknowns("meerkat", false) returns -1; but compareUnknowns(true, false) returns 1, which violates transitivity. If in doubt, try using createNullSafe(compareStrings), which can safely and consistently sort anything.
